Page Wrapper

Instead of using the Body to establish the basis for a page, we create a Page Wrapper.

Responsibilities

  • Background elements of the site: color, images, gradients, etc.
  • Establishing a min-height of 100vh. This ensures site-wide background elements cover the entire screen

CONTAINERS

Also called a section, the container is a div that vertically separates different areas on the page based on their purpose or topic. It can allow different background colors and properties for each section that help users associate content within it as belonging to the same component.

Responsibilities

  • Background elements of the section
    i.e. a header that has a different background color from the page wrapper, with a border underneath
  • Helps with the responsibilities of the Row by aligning it's children to the center

ROW

The Row

A div with a universal class name, typically called "row." It's sole purpose is establish the maximum width the majority of content will sit within. It prevents the content from reaching the edges of the screen on larger screens and disrupting the intended layout.
‍
Multiple Rows can be added to a Container to stack content vertically. If the content doesn't relate to each other, though, it might be better to use a new container instead.

Responsibilities

  • Establish and standardize the maximum content width for the entire site
  • Combo classes can be added to change the alignment of the columns within

Columns are divs that live directly inside rows. They allow you to separate content out horizontally. By default, they'll take up space evenly with sibling columns, but can have combo classes added to change their behavior and achieve more complex horizontal layouts.

Responsibilities

  • Separate content horizontally within a row
